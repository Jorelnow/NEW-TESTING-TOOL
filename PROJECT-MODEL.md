# Project Model

## Purpose
Define how shared work should be organized once multiple humans and agents are collaborating.

## Project fields
- name
- objective
- members
- linked repos
- linked deployments
- linked inboxes
- linked briefs
- risk level
- current priorities

## Example project types
- Market Ops
- Product Launch
- Research Sprint
- Creator Pipeline
- Joint Venture / Team Workspace

## Minimum useful views
- overview
- tasks
- agent assignments
- repo/deploy status
- brief history
- approvals queue

## Recommended first projects
- Market Ops HQ
- Creator Ops
- Build + Launch
